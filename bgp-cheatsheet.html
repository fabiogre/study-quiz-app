Kurzfassung Short version

BGP tauscht Routeninformationen aus, bewertet Pfade ueber Attribute und wird mit MP-BGP zum Steuerungsprotokoll fuer VPN-Dienste. Sobald iBGP zu gross wird, loesen Route Reflectors das Full-Mesh-Problem, brauchen aber eigene Loop-Protection ueber ORIGINATOR_ID und CLUSTER_LIST.

BGP exchanges route information, evaluates paths using attributes, and with MP-BGP becomes the control protocol for VPN services. Once iBGP grows too large, route reflectors solve the full-mesh problem, but they need dedicated loop protection via ORIGINATOR_ID and CLUSTER_LIST.

Route Reflectors

Route reflectors

  • loesen das iBGP-Full-Mesh-Problem
  • Rollen: `RR`, `Client`, `Non-Client`
  • Cluster = RR + Clients
  • redundante RRs im selben Cluster teilen typischerweise dieselbe Cluster-ID
  • solve the iBGP full-mesh problem
  • roles: `RR`, `Client`, `Non-Client`
  • cluster = RR + clients
  • redundant RRs in the same cluster typically share the same cluster ID

RR Update-Regeln

RR update rules

  • von Non-Client: zu Clients, nicht zu anderen Non-Clients
  • von Client: zu Clients und Non-Clients
  • von eBGP: zu Clients und Non-Clients
  • eBGP-Weitergabe bleibt policy-abhaengig
  • from non-client: to clients, not to other non-clients
  • from client: to clients and non-clients
  • from eBGP: to clients and non-clients
  • eBGP propagation stays policy-dependent

RR Loop Protection

RR loop protection

  • `ORIGINATOR_ID` = urspruenglicher iBGP-Originator
  • `CLUSTER_LIST` = durchlaufene RR-Cluster
  • eigene ID wieder gesehen = Route verwerfen
  • `CLUSTER_LIST` ist intra-AS, kein Ersatz fuer `AS_PATH`
  • `ORIGINATOR_ID` = original iBGP originator
  • `CLUSTER_LIST` = traversed RR clusters
  • seeing your own ID again = discard the route
  • `CLUSTER_LIST` is intra-AS, not a replacement for `AS_PATH`
